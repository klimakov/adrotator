# UI/UX ревью админки AdRotator

## Общая оценка
Интерфейс структурирован, стиль единообразный (Tailwind, indigo-акценты). Ниже — замечания и рекомендации по улучшению.

---

## 1. Навигация и layout

| Проблема | Рекомендация |
|----------|--------------|
| Иконки в сайдбаре — Unicode (◻ ▶ ◼ ⊞), выглядят по-разному в шрифтах | Заменить на SVG-иконки или один иконочный шрифт для единого стиля |
| Нет хлебных крошек при «Кампании → Редактировать» | Добавить breadcrumbs: Кампании / Название кампании |
| Сайдбар фиксированной ширины, на мобильных съедает место | На узких экранах: бургер-меню или сворачиваемый сайдбар |

---

## 2. Контент и таблицы

| Проблема | Рекомендация |
|----------|--------------|
| Таблицы с 8+ колонками на мобильных уезжают за экран | Обернуть таблицу в `overflow-x-auto` и при необходимости задать min-width |
| Статусы в ячейках на английском (active, paused, archived) | Показывать локализованные подписи: «Активна», «Пауза», «Архив» |
| Действия «Изменить» / «Удалить» только текстом | Добавить иконки или чётко разделить вторичное и опасное действие (удаление — красная кнопка/иконка) |
| Нет сортировки по колонкам | При росте числа записей — сортировка по дате, названию, показам |

---

## 3. Пустые состояния

| Проблема | Рекомендация |
|----------|--------------|
| «Нет кампаний. Создайте первую!» без действия | Добавить кнопку «Создать кампанию» прямо в пустом состоянии |
| То же для креативов и площадок | Аналогично: CTA «Новый креатив» / «Новая площадка» в блоке пустого состояния |

---

## 4. Формы

| Проблема | Рекомендация |
|----------|--------------|
| Ошибки API (сеть, 4xx/5xx) не показываются пользователю | Показывать toast или блок с текстом ошибки над формой |
| Кнопка «Отмена» визуально слабая | Оформить как вторичную кнопку (border, hover), не только цвет текста |
| Нет подсказок к полям (Zone Key, вес ротации) | Добавить короткий hint под полем или в placeholder |
| В форме креатива при загрузке файла нет ограничения размера в UI | Указать «до 10 МБ» рядом с кнопкой загрузки |

---

## 5. Обратная связь и микровзаимодействия

| Проблема | Рекомендация |
|----------|--------------|
| После «Сохранить» / «Создать» — только редирект, успех не виден | Краткий toast «Сохранено» или сообщение на странице списка |
| «Скопировать код» на форме площадки без подтверждения | Показать «Скопировано» на 2–3 сек или сменить текст кнопки |
| Загрузка — только текст «Загрузка...» | Добавить спиннер или скелетон карточек для единообразия |

---

## 6. Дашборд

| Проблема | Рекомендация |
|----------|--------------|
| Цветная точка в карточках визуально не привязана к числу | Вариант: цветная полоска слева у карточки или убрать точку |
| График «Показы за 14 дней»: подписи дат с rotate могут обрезаться | Проверить overflow; при необходимости укоротить формат (например, 7 фев) |
| Нет оси Y и масштаба | Подписать максимум показов (например, справа или над столбцами) |
| Нет выбора периода | В перспективе: 7 / 14 / 30 дней |

---

## 7. Доступность (a11y)

| Проблема | Рекомендация |
|----------|--------------|
| У кнопки удаления нет aria-label | Добавить `aria-label="Удалить кампанию Название"` |
| Связка label + input не везде через `htmlFor` / `id` | У всех полей форм задать `id` у input и `htmlFor` у label |
| Контраст «Отмена» (text-gray-500) на gray-50 | Проверить по WCAG; при необходимости усилить цвет |

---

## 8. Мелкие замечания

- **Сниппет:** моноширинный блок с тёмным фоном читаем; кнопка «Скопировать» логично рядом — остаётся только фидбек «Скопировано».
- **Единый стиль карточек:** `rounded-xl shadow-sm border` используется последовательно — ок.
- **Язык:** интерфейс на русском, только статусы в БД/API на английском — в UI показывать русские подписи.

---

## Приоритеты внедрения

1. **Высокий:** пустые состояния с CTA, обёртка таблиц `overflow-x-auto`, статусы на русском, фидбек «Скопировано», связка label/input в формах.
2. **Средний:** отображение ошибок при сохранении, кнопка «Отмена» как вторичная, спиннер/скелетон загрузки.
3. **Низкий:** breadcrumbs, иконки в навигации, сортировка таблиц, доработка графика.

В следующем шаге можно внедрить пункты из блока «Высокий» в коде.
